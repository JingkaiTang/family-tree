# 恢复默认布局设计

**日期：** 2026-07-15

**状态：** 产品方案已批准，等待实现计划

**关联设计：** `2026-07-11-family-unit-grid-layout-design.md`

## 1. 背景

当前同代家庭拖拽会把新的家庭顺序写入 `layoutPreferences.rowOrders`，画布缩放和平移则会保留在当前 UI 会话中。用户缺少一个明确的方式撤销这些交互影响，重新查看排版算法在当前家族数据上的默认结果。

本功能在树页面顶部增加“恢复默认布局”按钮。一次操作同时恢复算法默认布局和默认画布视图，不修改任何家族事实或语义关系。

## 2. 已确认的产品决策

1. 按钮放在树页面顶部操作区，靠近“清除视角”。
2. 按钮文案为“恢复默认布局”。
3. 点击后不弹确认框，因为被清除的只是可重新生成的布局偏好。
4. “全部重置”包含：
   - 清除用户拖拽形成的同代家庭顺序；
   - 使用当前数据重新运行默认布局算法；
   - 把缩放恢复为 `100%`；
   - 把画布平移恢复为 `(0, 0)`。
5. 没有手动同代顺序时按钮禁用。
6. `rowOrders` 的清除进入现有项目自动保存流程；画布视图只更新当前 UI 会话。

## 3. 重置边界

### 3.1 清除的状态

- `layoutPreferences.rowOrders`：当前家庭单元拖拽唯一写入的默认布局覆盖项。
- 当前 UI 会话中保留的画布缩放和平移状态。
- `FamilyCanvas` 内尚未完成的拖拽预览、待提交落点和旧布局恢复任务，防止它们在重置后重新写回旧顺序或旧场景。

### 3.2 保留的状态

- 全部人物、配偶、亲子和其他关系事实。
- 家庭颜色分配 `familyAccentAssignments`。
- 当前选中的人物。
- 当前视角人物；重置完成后不自动把该人物重新居中。
- 子女主要家庭归属 `childLayoutAssignments`，因为它表达多父母、多家庭场景中的语义选择，不是坐标拖拽结果。
- 为兼容旧数据而保留、但当前引擎不消费的历史坐标字段；本次功能不扩大为旧数据迁移。

## 4. 交互与状态流

```text
用户点击“恢复默认布局”
  → TreeView 清除 rowOrders
  → TreeView 清除当前 UI 会话的画布视图
  → TreeView 递增一次性 layoutResetVersion
  → FamilyCanvas 取消未完成的拖拽/恢复状态
  → FamilyCanvas 不携带 previousScene 发起全新布局
  → 仅最新布局请求可以提交场景
  → DOM 更新完成
  → PanZoomWrapper 强制恢复 scale=1、pan=(0,0)
  → 正常自动保存
```

`layoutResetVersion` 是一次性命令序号，不是持久化业务状态。它避免用组件整体重挂载来清理状态，也能让 `FamilyCanvas` 明确区分普通数据更新与用户主动重置。

## 5. 组件职责

### 5.1 Family Store

新增单一动作 `clearRowOrderPreferences()`：

- 当 `rowOrders` 已为空时直接返回，不制造无意义的脏状态。
- 否则只把 `rowOrders` 设为空数组，并标记数据待保存。
- 不重建整个 `layoutPreferences`，避免误删颜色和语义归属。

### 5.2 TreeView

- 通过 `rowOrders.length > 0` 计算按钮是否可用。
- 点击时依次清除 row order、清除 UI 会话中的画布视图，并递增 `layoutResetVersion`。
- 将 `layoutResetVersion` 作为属性传给 `FamilyCanvas`。
- 不改变选中人物和视角人物。

### 5.3 FamilyCanvas

- 普通数据变化继续沿用增量布局，可使用 `previousScene` 保持视觉稳定。
- 检测到新的 `layoutResetVersion` 时进入一次性重置流程：
  - 清除当前拖拽状态、预期行更新和待恢复任务；
  - 使进行中的旧异步布局请求失效；
  - 以当前已清除 `rowOrders` 的数据运行全量布局，且不传 `previousScene`；
  - 跳过视角人物自动聚焦；
  - 只在该请求仍是最新请求且场景已完成渲染后重置画布视图。

这样可以避免两个竞态：旧布局晚返回覆盖默认布局，以及默认视图刚恢复又被视角人物自动聚焦覆盖。

### 5.4 PanZoomWrapper

新增语义明确的 `resetToDefaultView()`，强制设置：

```ts
scale = 1
x = 0
y = 0
```

不复用现有 `resetView()` 的初始值语义。现有方法可能回到组件创建时读入的会话视图，不保证是产品定义的默认视图。

## 6. 异步与边缘场景

### 6.1 布局计算尚未结束

布局继续使用递增请求 ID。重置产生新请求后，所有更早请求即使晚返回，也不能写入场景或触发视图变化。

### 6.2 刚拖放完立即重置

Store 中的 `rowOrders` 先被清空；Canvas 同时丢弃本地待提交拖拽状态。任何旧落点结果都不能在重置后重新应用。

### 6.3 当前存在视角人物

保留视角人物身份，但本次重置请求跳过自动居中。之后用户主动切换视角人物时，既有聚焦行为照常工作。

### 6.4 数据本身发生变化

恢复的是“当前家族事实的算法默认布局”，不是历史快照。新增、删除或修改关系后点击重置，布局可以与最初打开页面时不同。

### 6.5 没有手动顺序但画布已缩放

按已确认规则，按钮只由手动同代顺序决定是否可用。单纯恢复缩放和平移继续使用画布已有的“重置”能力；顶部按钮表达的是撤销手动布局并进行全部重置。

## 7. 验收标准

1. 手动拖动家庭后，顶部“恢复默认布局”按钮可用。
2. 点击后 `rowOrders` 为空，页面按当前数据重新得到与无偏好输入一致的默认场景。
3. 点击后缩放为 `1`，平移为 `(0, 0)`，不会再次跳到视角人物。
4. 人物、关系、家庭颜色、选中人物、视角人物和 `childLayoutAssignments` 均保持不变。
5. 重置过程中更早的异步布局结果不能覆盖新场景。
6. 刚拖放完立即重置，不会出现旧顺序回写。
7. `rowOrders` 的清除进入项目自动保存；当前 UI 会话中的画布视图立即变为默认值。
8. 没有 `rowOrders` 时按钮禁用，点击不产生保存写入。

## 8. 测试策略

- **Store 单元测试：** 只清空 `rowOrders`、保留其他布局偏好、空状态不标脏。
- **TreeView 组件测试：** 按钮启用/禁用；点击调用清理动作、清除画布视图并发出新重置序号。
- **FamilyCanvas 组件测试：** 重置请求不使用旧场景；取消旧拖拽状态；忽略过期异步结果；提交最新场景后重置视图；重置期间不聚焦视角人物。
- **PanZoomWrapper 单元测试：** 无论组件创建时读入的会话视图为何，`resetToDefaultView()` 都得到 `1 / 0 / 0`。
- **回归测试：** 普通数据更新仍保持增量布局，现有视角聚焦和画布底部重置行为不变。

## 9. 实施约束

- 不通过重挂载整个 `FamilyCanvas` 实现重置，避免不必要的闪烁和状态丢失。
- 不修改排版算法本身；本功能只改变输入偏好和重算方式。
- 不引入新的持久化字段或依赖。
- 改动限定在 Store、TreeView、FamilyCanvas、PanZoomWrapper 及对应测试。
