## 概述

重新设计主角视角布局算法，解决当前实现的严重问题：布局混乱、层次不清、节点重叠。

采用**混合方案**：结合层次化布局和环形放射状布局的优点。

## 设计目标

- 清晰的关系距离层次（从内到外区分亲疏）
- 关系距离可视化（关系近的人在身边，远的人在外围）
- 紧凑不重叠（节点不重叠，合理利用空间）
- 美观的连线（连线不交叉，路径清晰）

## 核心算法

### 1. 关系距离计算

从主角出发 BFS，计算到每个人的最短关系步数：

- 配偶 = 1 步
- 父母/子女 = 1 步
- 兄弟姐妹 = 2 步
- 祖父母/孙辈 = 2 步
- 叔伯/堂表 = 3 步
- ...以此类推

### 2. 按距离分层

- 第 0 层：主角（距离 = 0）
- 第 1 层：配偶、父母、子女（距离 = 1）
- 第 2 层：兄弟姐妹、祖父母、孙辈（距离 = 2）
- 第 3 层：叔伯、堂表、曾祖辈（距离 = 3）
- ...以此类推

### 3. 每层内用 ELK 排列

同一层的节点用 ELK `force` 布局排列：
- 保证节点不重叠
- 间距自动计算
- 配偶会合并为同层虚拟节点后再拆回两个真实节点

### 4. 环形坐标计算

- 主角在中心（第 0 层，半径 = 0）
- 第 1 层半径 = BASE_RADIUS（10 单位）
- 每增加一层，半径增加 RING_GAP（8 单位）
- 每层内节点按 ELK 排列结果分配角度

### 5. 连线生成

当前实现手工生成连接线：
- **配偶连线**：连接两个配偶节点
- **父母→子女连线**：从父母中点连接到子女节点
- **兄弟姐妹连线**：通过父母节点间接连接（不直接连线）
- 尚未做全局自动避让，连线交叉需要视觉验收继续确认

## 数据流

```
主角 ID → BFS 计算距离 → 按距离分层 → 每层用 ELK 排列 → 计算环形坐标 → 生成连线
```

## 状态管理

- 现有 `viewpointId`（计算称谓用）保持不变
- 新增 `centerLayoutId`（控制布局模式）
- 两者完全独立

## UI 交互

- 选中某成员后，点击"以选中为中心布局"按钮
- 切换到主角视角布局，以该成员为中心
- 进入中心布局后显示"退出中心布局"按钮，点击后回到传统布局
- 选中不同成员后点击按钮 → 切换到新成员为中心的布局
- 按钮位置：在现有"以选中为视角"按钮旁边
